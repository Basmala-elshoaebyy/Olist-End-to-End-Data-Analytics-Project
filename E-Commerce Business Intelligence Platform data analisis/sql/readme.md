## SQL Analysis

SQL was used to validate the dataset, perform business analysis, and answer key operational questions.

### Revenue Analysis

* Calculated total business revenue.
* Identified top-performing product categories.
* Analyzed revenue distribution across states.

### Customer Analysis

* Determined regions with the highest order volume.
* Measured customer value and purchasing activity.

### Product Analysis

* Identified best-selling categories.
* Evaluated product contribution to overall revenue.

### Delivery Analysis

* Measured average delivery time.
* Compared logistics performance across states.

### Key SQL Concepts Demonstrated

* SELECT
* WHERE
* GROUP BY
* ORDER BY
* Aggregate Functions (SUM, AVG, COUNT)
* DISTINCT
* Business KPI Calculations

### Advanced SQL Joins

Although the final analytical dataset (master4) was created using Python and Pandas, additional SQL JOIN queries were developed to demonstrate understanding of relational database concepts.

The queries show how customer, order, product, and sales data can be combined using primary and foreign key relationships to answer business questions regarding revenue, customer behavior, and product performance.

Concepts demonstrated:

* INNER JOIN
* Primary Keys
* Foreign Keys
* GROUP BY
* ORDER BY
* Aggregate Functions
* Business KPI Calculation
